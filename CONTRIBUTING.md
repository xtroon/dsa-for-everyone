# Contributing to DSA Practice Vault

We love your input! We want to make contributing to this project as easy and transparent as possible.

## How to Contribute

1. **Fork the repo** and create your branch from `main`.
2. **Clone** your forked repository to your local machine.
3. **Add your changes** or new DSA problem solutions.
4. **Test your code** to ensure it compiles and runs correctly.
5. **Create a Pull Request** to the main repository.

## Adding New Problems
If you are adding a new DSA problem solution:
- Place it in the appropriate category folder (e.g., `array/`, `linked-list/`).
- Include the problem statement or a link to it at the top of the file.
- Include a brief explanation of your approach.
- State the **Time** and **Space Complexity** in the comments.
- Write clean and readable code with meaningful variable names.

## Community Guidelines

By participating in this project, you agree to be welcoming, inclusive, and respectful to all contributors. We encourage a collaborative and friendly environment where learners can grow together.

## 🐛 Reporting Issues and Bugs
If you find a bug or an issue with an existing solution (e.g., incorrect logic, suboptimal complexity, or compilation errors):
1. **Search existing issues** to check if the problem has already been reported.
2. **Open a new issue** if your problem is new.
3. Clearly describe the problem, include the file name, and detail the expected vs. actual behavior. Providing a test case that fails is highly appreciated!

## 💡 Suggesting Enhancements
You don't need to find a bug to contribute! We always welcome suggestions for improvements:
- Propose new, more optimized approaches for existing problems.
- Suggest new categories or fundamental DSA topics that should be covered.
- Open an issue to discuss your idea before writing code. This helps ensure your effort aligns with the project's goals.

Happy coding! 💻🔥
