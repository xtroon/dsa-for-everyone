## 🛠️ VS Code Setup & Execution

Since this repository is managed locally inside VS Code, here are some helpful tips for running the C++ files:

### Prerequisites

1. **Compiler**: Ensure you have a C++ compiler installed (e.g., `g++` via MinGW for Windows).
2. **Extensions**: Install the **C/C++** extension by Microsoft and **Code Runner** for easier execution.

### Compiling and Running

To manually run any file (e.g., `duplicate.cpp` inside the `array` folder) from the VS Code terminal:

```bash
# Navigate to the folder (e.g., array)
cd array

# Compile the C++ file
g++ duplicate.cpp -o duplicate

# Run the executable
.\\duplicate.exe
```

_Alternatively, click the **Run** button at the top-right corner if you have Code Runner installed._

## 🧠 DSA Fundamentals for Beginners

If you are new to Data Structures & Algorithms, here are some core concepts to understand before jumping into problems:

### 1. Time & Space Complexity (Big O Notation)
- **Time Complexity**: How much time an algorithm takes to run as the input size grows. (e.g., O(1), O(n), O(n^2))
- **Space Complexity**: How much extra memory an algorithm uses as the input size grows.
- **Why it matters**: It helps you compare different solutions and choose the most efficient one.

### 2. Core Data Structures
- **Arrays & Strings**: Sequential collections of elements. Great for fast lookups.
- **Linked Lists**: Nodes connected by pointers. Good for dynamic resizing and fast insertions/deletions.
- **Stacks & Queues**: LIFO (Last In First Out) and FIFO (First In First Out) structures.
- **Trees & Graphs**: Hierarchical and relational data representations.
- **Hash Tables (Maps/Sets)**: Key-value stores for blazing fast O(1) lookups.

### 3. Essential Algorithms
- **Sorting**: Bubble, Merge, Quick, etc.
- **Searching**: Linear Search, Binary Search (crucial for sorted data).
- **Recursion**: A function calling itself to solve smaller sub-problems.
- **Two Pointers & Sliding Window**: Common techniques for array/string problems.
